# Tugas 2: Membangun Ulang Halaman Landing dari Figma

Selamat datang pada pekan kedua program mentoring ini. Pada tahap ini, Anda akan ditantang untuk mengubah desain visual dari Figma menjadi sebuah halaman web yang fungsional dan responsif.

Tujuan dari tugas ini adalah untuk memperdalam pemahaman Anda mengenai tata letak modern menggunakan CSS serta alur kerja kolaborasi menggunakan Git.

## Langkah-Langkah Pengerjaan

### 1. Analisis Desain di Figma
Langkah pertama adalah membuka tautan Figma yang telah disediakan. Perhatikan elemen-elemen seperti tipografi, palet warna, dan jarak antar elemen. Identifikasi bagian mana saja yang akan menggunakan struktur Flexbox atau Grid. Pastikan Anda memahami perbedaan hierarki visual antara tampilan desktop dan mobile.

Silakan akses tautan desain melalui tautan berikut:
[Tautan Desain Figma - Challenge 2](https://www.figma.com/design/V8GNQPn1cu1NknDYWKrE49/Challenge-2?node-id=23413-8791&t=ILmynFv6W7ZC9q5j-1)

### 2. Persiapan Lingkungan Kerja dan Git
Buatlah sebuah folder baru untuk proyek ini. Lakukan inisialisasi Git pada folder tersebut menggunakan perintah git init. Sesuai dengan ketentuan, Anda wajib memiliki minimal dua branch, yaitu main dan feat/layout. Mulailah pengerjaan awal pada branch feat/layout untuk menjaga kerapihan riwayat commit Anda.

### 3. Pembuatan Struktur HTML
Susunlah kode HTML dengan memperhatikan elemen semantik seperti header, main, section, dan footer. Hal ini sangat penting untuk aksesibilitas dan struktur dokumen yang baik. Fokuslah pada penyusunan kerangka konten terlebih dahulu sebelum masuk ke tahap pemberian gaya.

### 4. Pemberian Gaya dengan CSS
Gunakanlah Flexbox atau CSS Grid untuk mengatur tata letak komponen utama dan pendukung. Hindari penggunaan posisi absolut secara berlebihan untuk elemen layout agar kode tetap fleksibel. Pastikan properti seperti margin, padding, dan ukuran font sesuai dengan spesifikasi yang ada di Figma.

### 5. Penerapan Desain Responsif
Terapkan media queries untuk memastikan halaman web Anda tampil optimal di berbagai ukuran layar. Perhatikan perubahan tata letak yang diperlukan saat berpindah dari tampilan mobile ke desktop atau sebaliknya. Lakukan pengujian secara mandiri dengan menggunakan fitur Inspect Element di browser Anda.

### 6. Pengelolaan Versi Kode dan Merge
Setelah semua fitur layout selesai diimplementasikan secara menyeluruh pada branch feat/layout, lakukan commit terhadap perubahan terakhir Anda. Selanjutnya, lakukan proses simulasi penggabungan atau merge dari branch feat/layout ke branch main. Pastikan tidak ada konflik pada kode sebelum proses penggabungan selesai.

## Kriteria Penilaian

Berikut adalah rincian parameter penilaian untuk tugas ini:

| Kriteria | Bobot | Deskripsi Singkat |
| :--- | :--- | :--- |
| Design Accuracy (Figma) | 30% | Tingkat kemiripan tampilan dengan desain Figma (layout, spacing, warna, typography). |
| Semantic HTML Structure | 25% | Penggunaan HTML yang semantik dan struktur yang benar. |
| CSS Implementation & Layout | 25% | Penggunaan CSS (selector, box model, flex/grid, spacing) secara benar dan efisien. |
| Responsiveness (Media Query) | 10% | Implementasi responsive sesuai breakpoint. |
| Code Quality & Git Workflow | 10% | Kerapian kode dan penggunaan Git yang benar. |

## Daftar Periksa Tugas

Berikut adalah kriteria minimal yang harus dipenuhi dalam tugas ini:

*   Halaman web bersifat responsif untuk perangkat mobile dan desktop.
*   Implementasi tata letak menggunakan metode Flexbox atau CSS Grid.
*   Setup Git menggunakan minimal dua branch, yaitu main dan feat/layout.
*   Melakukan simulasi pull request atau penggabungan branch secara benar.
